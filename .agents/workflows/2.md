## 📌 固定角色設計規範（每次必須帶入，不得省略）

使用者固定使用 **ChatGPT (DALL-E 3)** 生成封面圖。每次產出視覺 Prompt 時，必須帶入以下兩個角色作為封面主角，戲服與背景依當日主題調整，角色外觀特徵不得改變：

### 角色 A：數據張博士
- 台灣男性，40 歲左右，知性氣質
- **固定外觀**：深色西裝或當日主題戲服、黑框圓眼鏡、手持平板或數據螢幕、神情冷靜自信、偵探姿態（微微低頭審視數據）
- **當日可調整**：戲服換成當日主題（如：賽車服、武士袍、西裝加偵探帽）、手指向當日核心個股數字

### 角色 B：熱血阿明（韭菜散戶）
- 台灣男性，30 歲左右，親切感
- **固定外觀**：寬鬆 T 恤或當日主題服裝（顏色帶韭菜綠元素）、頭上可搭配韭菜造型裝飾、誇張的表情（依當日情緒：驚嚇 / 茫然 / 崩潰 / 興奮）
- **當日可調整**：服裝主題跟博士同步、表情依當日散戶心理設定

## 執行步驟

1. 找出 `reports/` 資料夾下，今天日期（或最近一次）生成的 `daily_report` 檔案，詳細閱讀其內容。

2. **【頻道敘事連貫性 + 觀看數回饋】** 讀取 `reports/06_YouTube_Assets/channel_story_log.md` 的最後 **7 行紀錄**，確認：
   - 上集（最近一次）推薦了哪支股票？預測方向是什麼？
   - 今日驗收結果如何（命中 ✅ / 失準 ❌）？
   - 根據結果，在標題中融入「延續感」設計：
     * 如果上集命中 → 使用「上集預言成真！」「驗證了！」等關鍵詞
     * 如果上集失準 → 使用「我上集預測錯了…」「比我想的更快！」等誠信型標題
     * 如果是第一次推薦無前例 → 正常設計懸念型標題
   - **【觀看數回饋】** 檢查最近 7 行中「觀看數」欄位（非 `-` 或 `待更新` 的數字）：
     * 找出**觀看數最高的那集**，記住它用的是哪種「使用標題」欄的標題類型（A/好奇心缺口、B/數據衝擊、C/利益相關）
     * 在步驟 4 生成標題時，**同類型標題排在第一位**，並在標題下方標注：`📊 參考依據：[日期] 標題[X] 獲 [N] 觀看（歷史最高）`
     * 如果所有觀看數都是 `-` 或 `待更新`，跳過此步驟，提示使用者上傳影片後在 channel_story_log.md 填入影片 ID

3. **【敘事框架輪換】** 讀取最近 3 份 `reports/06_YouTube_Assets/youtube_assets_*.md` 檔案，找出它們各自使用的敘事框架標籤（見下表），然後從**尚未在最近 3 集出現的框架**中，選出最符合今日報告數據特性的一種。

   **五種敘事框架（擇一使用）：**

   | 框架 ID | 名稱 | 適用情境 | 標題感覺 |
   |---|---|---|---|
   | `F1` | **角色研究型** | 今日有單一個股表現極異常（如外資狂買但股價跌） | 像人物紀錄片，聚焦「這公司到底是誰」 |
   | `F2` | **外資 vs 散戶對決型** | 法人與散戶行為明顯相反 | 對抗感強，「誰會贏？」 |
   | `F3` | **歷史重演型** | 今日圖形/行為與過去某集推薦的個股高度相似 | 懸念 + 驗證感，「這個圖我見過」 |
   | `F4` | **主題押注型** | 宏觀數據（商品、匯率、ETF）有重大異動 | 宏觀視野，「AI 記憶體時代要來了嗎？」 |
   | `F5` | **反直覺型** | 今日最大新聞表面看起來是利多/利空，但數據指向相反 | 衝擊感，「為什麼你不該在這時候追漲」 |

   選定後，在輸出檔案的標題下方標注：`**本集敘事框架：[框架ID] [框架名稱]**`，供下次輪換時參考。

   整份影片的**標題、說明欄、Hook 腳本**都必須符合選定框架的敘事邏輯。

4. 設計 2 組適合 YouTube 演算法且具備高點擊率 (CTR) 的 A/B 測試標題，其中**至少 1 組**帶有頻道連貫性敘事元素，且**所有標題都符合選定的敘事框架**。


4. 根據報告的市場氛圍與使用者指定的視覺風格，產出 **2 組 DALL-E 3 封面圖 Prompt**。

   🚨【強制規則，每次都必須遵守，不得省略】：
   - 每組 Prompt 必須讓**數據張博士（角色A）與熱血阿明（角色B）同時出現**在畫面中
   - 風格以使用者當日指定為主；未指定時預設使用賽博龐克或偵探電影風
   - 所有視覺與排版指令**合併寫進同一個 Prompt 區塊**，讓使用者可以直接複製貼給 ChatGPT

   每個 Prompt 必須包含：
   - **角色A描述**（帶入上方固定外觀 + 當日主題戲服）
   - **角色B描述**（帶入上方固定外觀 + 當日情緒表情）
   - **場景與背景美術風格描述**（當日主題）
   - **畫面文字指定**（明確告訴 ChatGPT 在畫面哪裡生成什麼繁體中文文字）

5. 撰寫以下完整的 YouTube 發布套件，全部寫進輸出檔案：

   **5a. 影片說明欄文案**
   - 開頭第一行：「📌 上集預測驗收：[填入上集核心標的與驗收結果]」
   - 接著是今日核心懸念、三個分析重點
   - 影片時間軸（根據 channel_story_log 的紀錄格式）
   - 社群 Hashtags（15個以內，含個股代號）

   **5b. 📌 置頂留言模板（每集必發）**
   格式如下，數字從今日報告精確提取：
   ```
   📌 今日核心數據快查
   🎯 觀察標的：[個股名稱 代號]
   💰 外資動向：[買超/賣超金額與排名]
   🛡️ 停損點：[具體價格與條件]
   📊 下集驗收：明日盤後見！
   ────────────────
   ⚠️ 本頻道僅供學習，不構成投資建議。請自行評估風險。
   ```

   **5c. ⏰ 建議上傳時間**
   台股收盤 1:30pm → 觀眾消化、下班通勤 → 最佳上傳窗口：
   **今日建議：[今天日期] 17:30–19:00**
   （若為週一，因為週末資訊積累，可提早至 17:00）

   **5d. 🎬 前 30 秒 Hook 腳本建議**
   根據今日最強異常，寫出 2–3 句「開場抓住觀眾」的 Hook 台詞，供博士在片頭使用。格式：
   - **Hook 句**：[懸念問句或衝突陳述，15字以內]
   - **數據炸彈**：[1個最衝擊的具體數字]
   - **承諾句**：「今天我們就來查清楚。」

   **5e. 📋 Playlist 系列追蹤建議**
   根據 channel_story_log 的追蹤紀錄，判斷是否有個股被連續推薦超過 2 集：
   - 如果有 → 建議建立「[個股名稱] 追蹤系列」播放清單，並寫出播放清單描述
   - 如果沒有 → 寫「暫無連續追蹤標的，持續觀察中」

   **5f. 📱 YouTube Community Post 文案**
   寫一則 150 字以內的社群貼文，適合在影片上傳前 30 分鐘發布：
   - 用今日最衝擊的一個數字當開頭
   - 結尾呼籲：「完整分析在今天的影片，留言告訴我你的看法！」
   - 加上 3–5 個 Hashtags

6. 使用 `write_to_file` 將以上 **5a–5f 的完整發布套件** 輸出並儲存到 `reports/06_YouTube_Assets/youtube_assets_[當天日期].md` 中。

7. 告知使用者素材已產生完畢，展示檔案路徑，並列出今日套件包含哪些項目。
